# Blazar 彩虹独角兽小马 mascot design

## Problem

Blazar 需要一个新的欢迎页 mascot。上一版方向过于严肃且在终端化实现后观感不佳，未能建立“可爱、轻快、愿意陪你开始工作”的第一印象。这一版需要明确转向更亲和、更讨喜、颜色更丰富的角色，同时避免重新滑向厚重科幻守卫风。

## Design goals

- 一眼读成可爱的角色，而不是符号化装饰
- 以“彩虹独角兽小马”为核心辨识点
- 色彩丰富但不刺眼，偏马卡龙软糖色
- 气质活泼调皮，但不吵闹、不喧宾夺主
- 与 Blazar 保持轻绑定，通过少量星光与科技点缀呼应品牌
- 优先适配欢迎页插画级表现，再考虑后续简化版本

## Non-goals

- 高冷、威严、神秘守卫式角色
- 盔甲、披风、武器、过重的科幻装置
- 纯表情包式夸张吵闹风格
- 高饱和霓虹彩虹导致的廉价感或视觉疲劳
- 为了终端 ASCII 而牺牲主形象可爱度

## Core concept

新版 mascot 的核心概念是 **星糖导航马**：一只 Q 版圆润、活泼调皮的彩虹独角兽小马，像会陪用户开始工作的轻快小搭子。

它不是“高冷 AI 化身”，而是“带着一点点星光和科技感的小迎宾”。用户看到它时，第一感受应当是可爱、轻松、愿意互动；第二层才读出它与 Blazar 的星际和智能体气质存在温和联系。

## Character read

- **角色类型：** 可爱小搭子 / 欢迎页迎宾 mascot
- **情绪关键词：** 活泼、调皮、机灵、讨喜
- **品牌关系：** 可爱优先，品牌点缀次之
- **整体优先级：** 可爱 > 识别度 > 品牌点缀 > 科技感

欢迎页里的情绪目标不是“庄重地等候命令”，而是“已经开心地等你开始今天的任务”。

## Silhouette and proportions

角色比例采用明显的 **Q 版圆润小马** 方向。

- 头部明显偏大，脸颊饱满
- 鼻口区短，减少写实马脸感
- 身体小巧、四肢短一些，整体像软糖或棉花糖般圆润
- 不追求复杂骨骼和真实马体结构
- 站姿或轻微侧身姿态应当清楚、稳定、容易记忆

整体轮廓要保证即使缩小观看，也能先读出“圆脑袋 + 独角 + 蓬松鬃毛”。

## Face and expression

表情需要非常亲和。

- 大而亮的眼睛，带星星状高光
- 不做拟人化真人眼线或复杂睫毛细节
- 眉眼关系可以偏灵动，形成“鬼马但不调皮过头”的表情
- 嘴部只需非常简洁的小笑意，不追求夸张张嘴表情

角色应该看上去像随时会开心地回应用户，但不会像卡通表情包一样失控夸张。

## Horn, mane, and tail

### Horn

- 独角较短，避免长而尖锐的攻击感
- 形状更像糖果旋角
- 颜色建议从奶黄到粉紫渐变

### Mane

- 鬃毛使用 **马卡龙彩虹**，核心颜色为粉、薄荷、天蓝、奶黄
- 层次要柔软、蓬松，不做尖锐火焰状分叉
- 彩虹色块要有节奏感，避免每一缕都过度抢戏

### Tail

- 尾巴应当像奶油云朵般蓬松
- 尾端可以附着少量星点闪光
- 动态上更适合轻轻弹动，而非大幅甩动

## Body and materials

- 身体底色以白色或极浅奶油白为主
- 表面质感偏柔和、干净、轻盈
- 不添加衣服、盔甲、披挂或手持物
- 身边可有 1-2 个极简漂浮星点，像 UI 粒子而非复杂装置
- 耳边可使用一个小星星发卡作为记忆点

这些元素应当共同服务于“软萌、轻盈、可记忆”，而不是堆叠世界观细节。

## Color system

推荐主配色：

- **主体底色：** 奶油白 / 很浅的暖白
- **彩虹主色：** 粉、薄荷、天蓝、奶黄
- **辅色：** 少量淡紫和蜜桃色
- **点缀色：** 小范围星光黄或浅青光点

色彩策略应当是“软糖彩虹”而不是“高饱和彩灯”。即使颜色很多，也必须保持舒服、甜和清爽。

## Welcome-screen behavior

欢迎页中的动态感应当轻快而克制。

### Initial state

角色略微侧头，像刚刚发现用户出现。

### Idle state

- 鬃毛或尾巴有轻微弹动感
- 身边星点缓慢漂浮
- 不做大幅度蹦跳或转圈

### Input state

- 眼神更专注一点
- 漂浮星点轻微聚拢
- 整体像在认真听用户输入内容

所有动势都应当是“轻、小、柔”，避免抢走欢迎文案和输入区的注意力。

## Rendering guidance

这版 mascot 的实现优先级必须明确：

1. **先做欢迎页主视觉插画**
2. **再考虑裁切版或头像版**
3. **最后才是高度简化的终端版**

如果后续需要做极简版，必须保留的识别点只有：

1. 圆脑袋
2. 糖果独角
3. 彩虹鬃毛轮廓

不要一开始就为了文本化渲染而牺牲主视觉质量。主视觉应先成立，简化版只负责延续记忆点。

## Acceptance criteria

设计成功的标准是：

- 用户第一眼就会说“这是只可爱的彩虹独角兽小马”
- 配色丰富但不刺眼，看上去讨喜
- 角色足够活泼调皮，但不破坏产品界面的稳定感
- 它与 Blazar 有轻度品牌联系，但不会重新变成沉重科幻 mascot
- 即便后续做简化版本，圆润脑袋、糖果独角和彩虹鬃毛仍然可辨认
